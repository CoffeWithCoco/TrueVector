Sending pace

How fast the test emails leave — paced to keep results comparable
Why pacing matters.
A brand-new sender that bursts ~29 odd messages at one mailbox is itself a suspicious pattern: the gateway can degrade the sender's reputation mid-run, so late emails get a worse verdict for reasons unrelated to their payload. Spacing the sends (with random jitter so it doesn't look like a cron) keeps reputation stable and verdicts comparable. The order is fixed: stealthiest techniques are sent first, the loud basic-detection ones (EICAR, macros) last, so a late reputation hit never reaches the subtle vectors. Set the interval to 0 for a fast burst (e.g. lab / GreenMail).
Base wait between each email. 0 = send back-to-back.
Random variation added to each interval. e.g. 90 ± 30 → 60–120s.
At 90 ± 30s, a full 29-technique run takes roughly 30–60 minutes. The campaign page updates live as each email is sent and analyzed.
A baseline canary is always sent first and last; this adds one every N techniques in between.

IMAP reader (verification)

Access to the receiving mailbox to read the test emails and measure what the gateway detected
MFA / Outlook / Microsoft 365
If the mailbox has MFA enabled you cannot use the regular password for IMAP. You have two options:
  • App Password — go to Account security → App passwords and generate a password specific to this app. Works on personal Microsoft accounts and on M365 when the admin allows it.
  • Corporate M365 without App Password — the admin must enable "IMAP Auth" in the EAC or via PowerShell: Set-CASMailbox -Identity user -ImapEnabled $true. Use the App Password or request an OAuth2 token (XOAUTH2).
  • Gmail — enable "App passwords" on the account (requires 2FA). Host: imap.gmail.com, port 993.

Microsoft 365 — Graph API (app-only)

Reads the mailbox via Microsoft Graph with no interactive login. For corporate M365 where you are admin.
Entra ID registration (one time, as admin)
  1. Entra ID → App registrations → New registration → copy the Application (client) ID and Directory (tenant) ID.
  2. Certificates & secrets → New client secret → copy the value.
  3. API permissions → Microsoft Graph → Application permissions → Mail.Read → Grant admin consent.
  4. Recommended: scope access to the test mailbox with New-ApplicationAccessPolicy in Exchange Online.
The mailbox to read is the Receiving mailbox above (target_email). It can be a shared mailbox (no license needed).
